# 게시판 프로젝트 요구사항

## 1. 프로젝트 개요

사용자가 회원가입과 로그인을 한 뒤 게시글과 댓글을 작성하고, 다른 사용자의 게시글에 좋아요를 누를 수 있는 REST API 기반 게시판 서버를 개발한다.

## 2. 프로젝트 목표

* 요구사항을 바탕으로 관계형 데이터베이스를 설계한다.
* REST 원칙을 고려하여 API를 설계한다.
* Spring Data JPA를 사용하여 데이터를 관리한다.
* Spring Security와 JWT를 사용하여 인증과 인가를 구현한다.
* QueryDSL을 사용하여 검색 및 동적 조회를 구현한다.
* 페이징을 적용하여 게시글 목록을 조회한다.
* 공통 API 응답과 전역 예외 처리를 구현한다.
* Swagger를 사용하여 API 명세를 제공한다.
* AWS S3를 사용하여 게시글 이미지를 관리한다.
* GitHub Actions와 AWS를 사용하여 서버를 배포한다.

---

# 3. 회원 기능

## 3.1 회원가입

* 사용자는 이메일, 비밀번호, 닉네임을 입력하여 가입한다.
* 이메일은 회원 간 중복될 수 없다.
* 닉네임은 회원 간 중복될 수 없다.
* 비밀번호는 암호화하여 저장한다.
* 가입한 사용자의 기본 권한은 `USER`이다.
* 탈퇴하거나 정지된 회원의 상태를 구분할 수 있어야 한다.

## 3.2 로그인

* 사용자는 이메일과 비밀번호로 로그인한다.
* 로그인에 성공하면 Access Token과 Refresh Token을 발급한다.
* 비밀번호가 일치하지 않으면 로그인이 실패한다.
* 탈퇴하거나 정지된 회원은 로그인할 수 없다.

## 3.3 토큰 재발급

* 유효한 Refresh Token을 제출하면 새로운 Access Token을 발급한다.
* Refresh Token이 만료되었거나 저장된 토큰과 일치하지 않으면 재발급에 실패한다.

## 3.4 로그아웃

* 로그아웃하면 저장된 Refresh Token을 삭제한다.
* 이후 해당 Refresh Token으로는 Access Token을 재발급할 수 없다.

## 3.5 내 정보 조회

* 로그인한 사용자는 자신의 회원 정보를 조회할 수 있다.
* 응답에 비밀번호를 포함하지 않는다.

# 4. 게시글 기능

## 4.1 게시글 작성

* 로그인한 사용자만 게시글을 작성할 수 있다.
* 게시글은 제목, 내용, 카테고리를 가진다.
* 제목은 비어 있을 수 없다.
* 내용은 비어 있을 수 없다.
* 존재하지 않거나 비활성화된 카테고리는 사용할 수 없다.
* 게시글에는 여러 장의 이미지를 첨부할 수 있다.

## 4.2 게시글 단건 조회

* 사용자는 게시글 하나의 상세 내용을 조회할 수 있다.
* 삭제된 게시글은 조회할 수 없다.
* 게시글 작성자 정보, 카테고리, 댓글 수, 좋아요 수를 반환한다.
* 로그인한 사용자가 해당 게시글에 좋아요를 눌렀는지 반환한다.
* 게시글을 조회하면 조회 수가 증가한다.

## 4.3 게시글 목록 조회

* 게시글 목록은 페이지 단위로 조회한다.
* 기본 정렬은 최신순이다.
* 다음 조건을 선택적으로 사용할 수 있다.

    * 제목 및 내용 검색
    * 작성자 닉네임 검색
    * 카테고리 필터
    * 최신순
    * 좋아요순
    * 조회순
* 삭제된 게시글은 목록에 표시하지 않는다.
* 목록 응답에서는 게시글 본문 전체를 반환하지 않는다.

## 4.4 게시글 수정

* 로그인한 사용자만 게시글을 수정할 수 있다.
* 게시글 작성자만 자신의 게시글을 수정할 수 있다.
* 관리자는 모든 게시글을 수정할 수 있다.
* 제목, 내용, 카테고리를 수정할 수 있다.

## 4.5 게시글 삭제

* 게시글 작성자만 자신의 게시글을 삭제할 수 있다.
* 관리자는 모든 게시글을 삭제할 수 있다.
* 게시글은 우선 논리 삭제한다.
* 삭제된 게시글은 일반 조회에서 제외한다.

# 5. 댓글 기능

## 5.1 댓글 작성

* 로그인한 사용자만 댓글을 작성할 수 있다.
* 존재하며 삭제되지 않은 게시글에만 댓글을 작성할 수 있다.
* 댓글 내용은 비어 있을 수 없다.

## 5.2 댓글 목록 조회

* 게시글별 댓글을 작성 순서대로 조회한다.
* 댓글 작성자의 닉네임을 반환한다.
* 삭제된 댓글은 목록에서 제외한다.

## 5.3 댓글 수정

* 댓글 작성자만 자신의 댓글을 수정할 수 있다.
* 관리자는 모든 댓글을 수정할 수 있다.

## 5.4 댓글 삭제

* 댓글 작성자만 자신의 댓글을 삭제할 수 있다.
* 관리자는 모든 댓글을 삭제할 수 있다.
* 댓글은 논리 삭제한다.

## 5.5 확장 기능

MVP가 완성된 후 댓글의 `parent_id`를 사용하여 대댓글을 추가한다.

# 6. 좋아요 기능

## 6.1 좋아요 등록

* 로그인한 사용자는 게시글에 좋아요를 등록할 수 있다.
* 한 사용자는 하나의 게시글에 한 번만 좋아요를 등록할 수 있다.
* 삭제된 게시글에는 좋아요를 등록할 수 없다.

## 6.2 좋아요 취소

* 좋아요를 등록한 사용자는 좋아요를 취소할 수 있다.
* 이미 취소된 좋아요를 다시 취소해도 데이터가 중복 변경되지 않아야 한다.

---

# 7. 카테고리 기능

* 사용자는 활성화된 카테고리 목록을 조회할 수 있다.
* 관리자만 카테고리를 생성하거나 수정할 수 있다.
* 사용 중인 카테고리는 바로 삭제하지 않고 비활성화한다.

초기 카테고리:

* 자유
* 질문
* 정보
* 공부 기록

---

# 8. 이미지 기능

* 로그인한 사용자만 이미지를 업로드할 수 있다.
* 이미지는 AWS S3에 저장한다.
* 데이터베이스에는 이미지 URL과 S3 Object Key를 저장한다.
* 게시글 하나에 최대 5개의 이미지를 첨부할 수 있다.
* 이미지 파일 크기와 형식을 검증한다.
* 허용 형식은 JPEG, PNG, WebP로 제한한다.
* 게시글 이미지가 제거되면 S3 파일도 삭제한다.

---

# 9. 관리자 기능

관리자는 다음 작업을 수행할 수 있다.

* 모든 게시글 삭제
* 모든 댓글 삭제
* 카테고리 생성 및 비활성화
* 회원 상태 변경

관리자 기능은 기본 게시판 기능이 완성된 후 구현한다.

# 10. 공통 정책

## 10.1 인증

* 회원가입, 로그인, 토큰 재발급, 게시글 조회 API는 인증 없이 호출할 수 있다.
* 게시글 작성, 수정, 삭제, 댓글, 좋아요, 이미지 API는 인증이 필요하다.

## 10.2 응답 형식

모든 API는 통일된 형식으로 응답한다.

```json
{
  "success": true,
  "code": "POST201",
  "message": "게시글을 작성했습니다.",
  "result": {
    "postId": 1
  }
}
```

## 10.3 예외 응답

```json
{
  "success": false,
  "code": "POST404",
  "message": "게시글을 찾을 수 없습니다.",
  "result": null
}
```

## 10.4 날짜

* 서버 내부 날짜는 `LocalDateTime`으로 관리한다.
* API에서는 ISO 8601 형식으로 반환한다.

## 10.5 삭제 정책

* 회원, 게시글, 댓글, 카테고리는 상태값을 사용하는 논리 삭제를 적용한다.
* 게시글 좋아요와 Refresh Token은 물리 삭제를 적용한다.
